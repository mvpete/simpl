## Making Your Contribution

### Step 1: Prepare Your Changes

1. **Create a branch:**
   ```bash
   git checkout -b lesson-contribution
   ```

2. **Make your changes:**
   - Edit files or add new ones
   - Test your changes thoroughly
   - Follow existing code style

3. **Test your contribution:**
   - If you added code, run it
   - If you added tests, verify they pass
   - If you edited lessons, read them carefully

### Step 2: Commit Your Work

```bash
# Add your changes
git add .

# Commit with a descriptive message
git commit -m "Add [description of your contribution]"

# Examples:
# git commit -m "Add quiz game example demonstrating objects and arrays"
# git commit -m "Improve L6 with additional multi-method examples"
# git commit -m "Add SIMPL style guide for best practices"
```

### Step 3: Push and Create Pull Request

1. **Push to your fork:**
   ```bash
   git push origin lesson-contribution
   ```

2. **Create a Pull Request:**
   - Go to your fork on GitHub
   - Click "Pull Request"
   - Select your branch
   - Write a clear description

3. **PR Description Template:**
   ```markdown
   ## Description
   Brief description of what you added/changed.
   
   ## Type of Contribution
   - [ ] New example program
   - [ ] Lesson improvement
   - [ ] New guide
   - [ ] Bug fix
   - [ ] Tests
   
   ## Testing
   How you tested your changes.
   
   ## Related Lessons
   Which lessons this relates to (if applicable).
   ```

### Step 4: Respond to Feedback

- Be open to suggestions
- Make requested changes
- Ask questions if unclear
- Be patient and professional

## Contribution Guidelines

### Code Style

**SIMPL Code:**
```simpl
# Good: Clear names, comments, proper spacing
def calculate_average(numbers) {
    let sum = 0;
    let count = length(numbers);
    
    for(let i = 0; i < count; i = i + 1) {
        sum = sum + numbers[i];
    }
    
    return sum / count;
}

# Avoid: Unclear names, no comments, cramped
def ca(n){let s=0;let c=length(n);for(let i=0;i<c;i=i+1){s=s+n[i];}return s/c;}
```

**Lesson Writing:**
- Use clear, simple language
- Include code examples for every concept
- Provide both explanation and code
- Use proper markdown formatting
- Test all code examples

## Example Contributions

### Example 1: New SIMPL Program

**File:** `examples/fibonacci.sl`
```simpl
###
### Fibonacci Sequence Generator
### Demonstrates: Recursion, functions, arrays
### Author: Alice Developer
###

@import io
@import array

# Recursive fibonacci
def fib_recursive(n) {
    if(n <= 1) {
        return n;
    }
    return fib_recursive(n - 1) + fib_recursive(n - 2);
}

# Iterative fibonacci (more efficient)
def fib_iterative(n) {
    if(n <= 1) {
        return n;
    }
    
    let a = 0;
    let b = 1;
    
    for(let i = 2; i <= n; i = i + 1) {
        let temp = a + b;
        a = b;
        b = temp;
    }
    
    return b;
}

# Generate fibonacci sequence
def fib_sequence(count) {
    let sequence = new [];
    
    for(let i = 0; i < count; i = i + 1) {
        push(sequence, fib_iterative(i));
    }
    
    return sequence;
}

# Demo
println("=== Fibonacci Numbers ===");
println("First 10 Fibonacci numbers:");

let fibs = fib_sequence(10);
println(fibs);

println("");
println("20th Fibonacci number: " + fib_iterative(20));
```

### Example 2: Lesson Improvement

**Addition to L7 (Arrays):**
```markdown
## Common Array Patterns

### Pattern: Map (Transform Each Element)

```simpl
@import io
@import array

def square(x) {
    return x * x;
}

def map(arr, func) {
    let result = new [];
    let count = length(arr);
    
    for(let i = 0; i < count; i = i + 1) {
        push(result, func(arr[i]));
    }
    
    return result;
}

let numbers = new [1, 2, 3, 4, 5];
let squares = map(numbers, &square);
println(squares);  # [1, 4, 9, 16, 25]
```

This pattern applies a function to each element and returns a new array.
```

# Debugging SIMPL Programs

## Common Errors and Solutions

### Error: "Undefined variable"

**Problem:** Using a variable before declaring it.

**Solution:**
```simpl
# Wrong
println(x);
let x = 10;

# Correct
let x = 10;
println(x);
```

### Error: "Function not found"

**Problem:** Missing `@import` directive.

**Solution:**
```simpl
# Wrong
println("hello");

# Correct
@import io
println("hello");
```

## Debugging Techniques

### 1. Print Debugging

Add `println` statements to see variable values:

```simpl
let x = calculate_something();
println("Debug: x = " + x);  # Check the value
```

### 2. Simplify

Comment out code to isolate problems:

```simpl
# Complex code
# let result = complex_function(a, b, c);

# Simplified version for testing
let result = 42;
```

### 3. Check Types

Verify variable types match expectations:

```simpl
def process(value is number) {
    # Only accepts numbers
}
```
```
