# Contributing to SIMPL

Thanks for contributing to SIMPL.

## Getting Started

1. Fork the repository.
2. Clone your fork.
3. Follow `lessons/GETTING_STARTED.md`.

## Contribution Types

- Bug fixes
- Lesson/documentation improvements
- New examples in `examples/`
- Tests in `simpl.test/`
- Language/runtime improvements

For beginner-friendly tasks, see `lessons/BEGINNER_ISSUES.md`.

## Recommended Workflow

1. Create a branch for your change.
2. Keep scope focused (one problem per PR).
3. Add or update tests where applicable.
4. Verify your change manually or with tests.
5. Open a pull request with a clear summary.

## PR Checklist

- [ ] Problem statement is clear
- [ ] Change is scoped and minimal
- [ ] Tests added/updated when behavior changed
- [ ] Documentation updated when needed
- [ ] Validation steps included in PR description

## Style Guidance

### C++
- Keep changes small and readable.
- Prefer explicit, maintainable code over cleverness.
- Avoid unrelated refactors in the same PR.

### Lessons/Docs
- Use clear language and runnable examples.
- Keep instructions step-by-step.
- Verify links and commands.

## Reporting Issues

When filing issues, include:
- what you expected
- what happened
- minimal reproduction snippet
- environment/tool versions

## Code of Conduct

Be respectful, constructive, and patient in reviews and discussions.
